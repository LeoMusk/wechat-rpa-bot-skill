# RPA 基本环境配置清单 (Basic Setup Checklist)

当日志无明显报错但 RPA 功能异常时，**必须引导用户按此清单逐项自查**。这些是最常见的隐性环境问题，不会在日志中报错，但会直接导致消息读取失败、自动回复失效或任务中断。

---

## 1. 微信客户端版本

**要求：必须使用 4.1.7 版本，且关闭自动升级。**

微信会在后台静默自我升级，导致 RPA 与新版本不兼容而失效，但日志不会提示版本问题。

**检查步骤：**
1. 打开微信 → 点击左下角头像 → **关于微信**，查看当前版本号。
2. 如果版本不是 `4.1.7`，需要卸载并重新安装指定版本。
3. 安装后，前往微信 **设置 → 通用 → 关闭"自动升级"**，防止再次被升级。

**4.1.7 安装包下载地址：**
https://n2b8xxdgjx.feishu.cn/wiki/Nbauw9HWsihsQ7kgjYPcfZSCnKb

> Agent 提示话术：「请您打开微信左下角头像 → 关于微信，确认版本是否为 4.1.7。如果不是，需要先卸载再按上方链接下载安装指定版本，安装完成后记得在设置→通用中关闭自动升级。」

## 2. 置顶会话数量过多导致新消息读取失败

**现象：有新消息但机器人始终读不到，消息未被回复。**

RPA 通过截图识别会话列表来读取新消息。若置顶会话过多，有新消息的会话可能被挤出可视区域，导致 RPA 始终读不到该会话的新消息。

**处理建议：**
1. 尽量减少置顶会话的数量，将不必要的置顶取消。
2. 将微信窗口高度拉到屏幕最大化，让每次截图尽可能覆盖更多会话条目。

> Agent 提示话术：「请检查微信会话列表中是否有大量置顶会话。置顶会话太多会导致有新消息的会话被推到可见区域以外，RPA 就无法读取到。建议取消不必要的置顶，并把微信窗口拉到最高。」

---

## 3. 屏幕息屏 / 睡眠导致 RPA 中断

**现象：RPA 启动后一段时间自动停止工作，或任务执行不完整。**

RPA 依赖屏幕截图和鼠标模拟操作，一旦屏幕息屏或系统进入睡眠，RPA 将无法正常工作。

**必须完成的设置：**
1. **关闭自动息屏**：系统设置 → 电源和睡眠 → 屏幕关闭时间设置为「从不」。
2. **关闭睡眠**：系统设置 → 电源和睡眠 → 睡眠时间设置为「从不」。
3. RPA 托管运行期间，**不要移动鼠标或操作键盘**，手动操作会干扰 RPA 的鼠标模拟，导致任务执行错误。

> Agent 提示话术：「RPA 运行期间需要保持屏幕常亮。请前往 Windows 系统设置 → 电源和睡眠，将屏幕关闭时间和睡眠时间都设置为"从不"。另外，RPA 自动操作时请不要移动鼠标，否则会干扰 RPA 的自动点击。」

---

## 4. 设备时间偏差过大导致消息处理异常

**现象：消息未被处理、定时任务触发时机不对，或消息处理顺序混乱。**

RPA 读取的是运行设备的本地系统时间。如果设备时间与真实网络时间相差超过 **3 分钟**，会导致消息时间戳判断错误，进而影响消息的处理逻辑（如误判为旧消息跳过、定时任务提前或延迟触发等），且日志中不会有明显报错。

**检查与修复步骤：**
1. 打开浏览器，搜索「北京时间」，对比当前网络时间与系统右下角时钟显示的时间。
2. 如果误差超过 3 分钟，需要同步系统时间：**系统设置 → 时间和语言 → 日期和时间 → 立即同步**（或开启"自动设置时间"）。
3. 同步完成后，确认系统时间与网络时间一致，再重新启动 RPA 托管。

> Agent 提示话术：「请对比一下您设备右下角的系统时间和浏览器搜索"北京时间"显示的时间，如果相差超过 3 分钟，请前往系统设置 → 时间和语言 → 日期和时间，点击"立即同步"或开启自动设置时间，然后重新启动 RPA。」

## Agent 使用指引

当用户反馈 RPA 异常，但你通过 `wechat_get_task_logs` 查看日志后**发现无明显报错**时，请按以下顺序引导用户自查：

1. 先确认微信版本是否为 `4.1.7`，是否关闭了自动升级。
2. 询问是否有大量置顶会话，微信窗口是否最大化。
3. 确认系统是否开启了息屏/睡眠，RPA 运行时是否有手动操作鼠标。
4. 检查设备系统时间与真实网络时间的误差是否超过 3 分钟，如有需立即同步。

将以上四项逐条发给用户，让用户对照自查，通常可以解决 80% 以上的「日志无报错但功能不正常」的问题。
