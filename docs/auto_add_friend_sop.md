# 自动加人操作指南与参数说明

本文档用于指导如何配置和排查「自动加人（根据名单批量添加微信好友）」功能的问题。当用户在开启自动加人功能时遇到报错，或者对功能配置及名单导入有疑问时，请参考此文档为用户提供解答和操作指导。

## 1. 核心运行逻辑说明
自动加人功能在启动时（`enabled: true`），系统会首先检查数据库中是否还有**待添加的好友名单**。
如果名单为空，接口将拒绝启动任务并返回如下报错：
`当前无可添加名单，请先导入名单`

此外，自动加好友任务没有强制的必传配置参数。系统会按照以下优先级获取配置：
1. 本次 API 请求传入的参数
2. 历史成功运行的缓存配置（`history_config`）
3. 系统默认的安全阈值配置

## 2. 接口配置参数 (settings) 详解

| 参数名 | 类型 | 含义说明 | 示例/安全建议 |
| :--- | :--- | :--- | :--- |
| **maxFriendsPerDay** | int | **每日最大添加数**：控制单个微信账号每天最多发送多少个好友请求。用于防范微信风控封号。 | 默认值 `10`。**强烈建议每天每个微信号加人不超过 10-15 个**。 |
| **maxProcessPerTime** | int | **单次执行添加数**：每次循环任务执行时，连续发送几个好友请求。 | 默认值 `1`。**推荐设置为 1-2 个**，避免短时间内高频操作。 |
| **checkInterval** | int | **循环间隔(分钟)**：执行完一次加人任务后，等待多久再执行下一次。 | 默认值 `60`。**推荐设置为 60 分钟**，模拟真人操作频率。 |
| **verifyMessage** | string | **验证消息**：加好友时附带的打招呼话术。 | 示例：`你好，我是xxx`（默认留空） |
| **multiCycleEnabled** | bool | **多微信循环**：如果登录了多个微信，为 `true` 则对指定账号循环分配加人任务；为 `false` 只使用当前活跃微信加人。 | 默认值 `false` |
| **accountIds** | array | **循环微信账号**：当 `multiCycleEnabled` 为 `true` 时，指定参与循环加人任务的微信 `account_id` 数组。 | `["wxid_xxx", "wxid_yyy"]` |

## 3. 常见报错处理 SOP（操作指导）

### 场景：用户点击开启自动加人报错，提示 `当前无可添加名单，请先导入名单`
**原因分析**：用户的加人任务队列是空的，系统不知道该去加谁，因此拒绝启动。必须先准备好目标人群的 Excel 名单并导入 RPA 系统。

**给用户的操作建议（话术参考）**：
“由于系统检测到您的**待添加好友名单为空**，所以无法启动自动加人任务。请按照以下步骤先导入名单：

**第一步：准备名单数据**
1. 请下载官方提供的 Excel 名单模板：[点击下载名单模板](https://healthyeveryday.oss-cn-shanghai.aliyuncs.com/file/%E5%90%8D%E5%8D%95%E6%A8%A1%E6%9D%BF%20.xlsx)
2. 打开模板后，请严格按照表头填写数据（单次最多导入 500 条）：
   - **微信号/手机号**（必填）：对方的联系方式
   - **备注**（选填）：通过后自动设置的微信备注名（例如：`Alex-朝总`）
   - **标签**（选填）：通过后自动打上的微信标签（例如：`VIP客户`）

**第二步：导入名单并启动**
1. 点击当前页面右上角的 **[管理 BOT]** 按钮，打开 RPA 详细配置后台。
2. 在后台界面中，切换到 **[自动化 SOP] -> [自动加好友]** 页面。
3. 找到导入按钮，上传您刚才整理好的 Excel 文件，系统会自动解析并加入任务队列。
4. 导入完成后，您可以在该页面调整加人频率（如每天10个，间隔60分钟），然后点击 **[启动]**。
5. 启动成功后，下次就可以直接在当前的快捷页面一键开启或关闭了！”
