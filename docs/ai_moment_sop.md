# AI 朋友圈操作指南与参数说明

本文档用于指导如何配置和排查「AI 朋友圈（自动点赞与评论）」功能的问题。当用户在开启 AI 朋友圈功能时遇到报错，或者对功能配置有疑问时，请参考此文档为用户提供解答和操作指导。

## 1. 核心运行逻辑说明
AI 朋友圈功能在启动时，会优先尝试读取用户最后一次成功配置的历史记录（`history_config`）。
如果在尝试开启时（`enabled: true`），用户既没有传入必填的配置参数，系统也找不到任何历史成功配置记录，接口将返回如下报错：
`开启任务失败：缺少必传参数 interactionMode 或 agentId，且未找到历史配置`

## 2. 接口配置参数 (settings) 详解

如果需要重新配置或了解各字段含义，请参考以下说明：

| 参数名 | 类型 | 含义说明 | 示例/可选值 |
| :--- | :--- | :--- | :--- |
| **commentLimit** | int | **总评论上限**：每次执行朋友圈自动化任务时，最多评论的好友总数。达到此数值后自动结束本次任务。 | 默认值 `100` |
| **perFriendLimit** | int | **单好友次数**：控制单次任务中，同一个好友最多评论的次数。防止连续刷屏。 | 默认值 `2`。如设为2，某好友连发3条，只评论前2条。 |
| **checkInterval** | int | **循环间隔(分钟)**：任务执行完后，等待多长时间再次执行下一次任务。 | 默认值 `120`（即2小时） |
| **reachLastPosition** | string | **达到上次位置时的处理方式**：当任务滚动到上一次任务已经处理过的位置时如何应对。 | `stop`：终止本次任务（推荐）<br>`continue`：跳过并继续 |
| **interactionMode** | string | **互动模式(必传)**：定义点赞和评论的具体行为策略。 | `like_only`：仅点赞（全点）<br>`comment_only`：仅评论（有评语才评，不点赞）<br>`like_and_comment`：评论才点赞（有评语才评并赞）<br>`like_always_and_comment`：点赞+评论（必点赞） |
| **agentId** | string | **智能体ID(必传)**：用于生成朋友圈评论的 AI 智能体 ID。 | （可根据用户绑定的智能体名称动态获取） |
| **selectedTags** | array | **指定好友标签**：只评论带有选中标签的好友朋友圈，其他人跳过。 | `[]`（默认，不限制） |
| **multiCycleEnabled** | bool | **多微信循环**：如果登录了多个微信，为 `true` 则对指定账号循环评论；为 `false` 只评论当前活跃微信。 | `false` |
| **selectedAccounts** | array | **循环微信账号**：当 `multiCycleEnabled` 为 `true` 时，指定需要循环的微信 `account_id` 数组。 | `["wxid_xxx", "wxid_yyy"]` |

## 3. 常见报错处理 SOP（操作指导）

### 场景：用户点击开启 AI 朋友圈报错，提示缺少 `interactionMode` 或 `agentId`
**原因分析**：这是因为用户是第一次使用该功能，或者之前清除了本地缓存，导致 RPA 服务没有找到该功能运行所必需的历史默认配置。

**给用户的操作建议（话术参考）**：
“由于您是首次启动 AI 朋友圈功能，系统还没有您的偏好配置记录。为了让 AI 知道如何帮您点赞和评论，需要您先手动进行一次初始化配置。
**请按以下步骤操作：**
1. 点击 RPA 页面右上角的 **[管理 BOT]** 按钮，这会打开 RPA 服务的详细配置后台。
2. 在后台界面中，找到并切换到 **[自动化 SOP]** 或 **[AI 朋友圈]** 配置页面。
3. 根据您的需要，完整设置一次点赞/评论策略（互动模式）、绑定的 AI 智能体等参数。
4. 在后台点击 **[启动]** 任务。
5. 启动成功后，系统会自动保存您的这套配置。下次您就可以直接在当前的快捷页面一键开启或关闭 AI 朋友圈了！”
